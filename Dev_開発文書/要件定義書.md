# 要件定義書 — 日本国憲法 電光掲示板

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | 日本国憲法 電光掲示板 |
| 目的 | 日本国憲法の条文を電光掲示板（LEDスクロール）風に表示するWebページを制作する |
| 公開方法 | GitHub Pages にデプロイして公開（PWA対応） |
| 対応デバイス | スマートフォン、タブレット、PC（レスポンシブ対応、ホーム画面インストール可） |

## 2. システム要件

### 2.1 技術スタック
- **フロントエンド**: HTML / CSS / JavaScript（バニラ、フレームワーク不使用）
- **フォント**: Google Fonts — Noto Sans JP (400, 700)
- **アクセス解析**: Google Analytics 4（gtag.js、測定ID: `G-8D75FXMHHD`、PWAバックグラウンド復帰時ビーコン送信、操作イベントビーコン送信。`start_scroll` には選択条文・メッセージに加え、文字サイズ・速度の設定値も記録する）
- **ホスティング**: GitHub Pages（静的サイト）

### 2.2 ブラウザ対応
- モダンブラウザ（Chrome, Safari, Firefox, Edge の最新版）
- CSS `:has()` セレクタ対応ブラウザ（2023年以降の主要ブラウザ）

### 2.3 ファイル構成
- `index.html` — メインHTMLファイル（CSS・JS インライン、単一ファイル完結、Google Analytics gtag.js 埋め込み）
- `manifest.json` — PWAマニフェスト（アプリ名・アイコン・テーマ色）
- `sw.js` — Service Worker（オフラインキャッシュ）
- `icon-192.png` / `icon-512.png` — PWAアイコン
- `日本国憲法.txt` — 憲法原文テキスト（データソース）
- その他画像・動画は説明用資材

## 3. 機能要件

### 3.1 画面構成
アプリケーションは2画面構成とする。

#### 3.1.1 設定画面（#settings）
ユーザーがスクロール表示のパラメータを設定する画面。

| 設定項目 | 選択肢 | デフォルト |
|----------|--------|-----------|
| 文字サイズ | 小(25vmin) / 中(40vmin) / 大(55vmin) | 中 |
| スクロール速度 | 緩(160px/s) / 中(300px/s) / 速(500px/s) | 中 |
| 文字色 | 10色（白・緑・黄・水色・橙・赤・ピンク・紫・青・ライム）からチェックボックスで**複数選択可** | 黄(#ffe600) |
| 条文選択 | 前文（4段落）＋全103条（項単位に分割、164エントリ）からチェックボックスで複数選択 | なし |
| メッセージ | 2欄（①②）のテキスト入力。ラベル（半角カナ「ﾒｯｾｰｼﾞ①」「ﾒｯｾｰｼﾞ②」）と入力欄を横並び配置。入力されると条文と同様にスクロール表示される | 空 |
| 縦置き | スライド式トグルスイッチ。ONで縦書き縦スクロールモードに切替 | OFF |

- 条文選択とメッセージ（①②いずれか）の少なくとも一方が必須（全て空の場合アラート表示）
- メッセージは入力のある欄のみ条文の先頭に配置（①が空で②のみ入力も可）。タイトルには「Msg1」「Msg2」と表示
#### 3.1.1a インストールバナー
設定画面の最下部（QRコードの下）にPWAインストール案内バナーを表示。

- 全デバイス共通で表示（PC/スマホ/タブレット）
- PWAとして起動中（`display-mode: standalone`）の場合は非表示
- ×ボタンで閉じた場合は24時間非表示（localStorage）
- Chrome/Edge: ネイティブインストールダイアログを呼び出し
- iOS Safari: 「共有→ホーム画面に追加」の手順を案内

#### 3.1.1b QRコード・キャッチコピー表示
設定画面の実行ボタン下部に、キャッチコピーとQRコードを配置。

- キャッチコピー:「Let's Enjoy Our 憲法 Together」（`#7a9a7a`・11px、opacity影響外）
- QRコード: `url.png` をBase64エンコードしてHTMLにインライン埋め込み（単一ファイル完結を維持）
- QRサイズ: 80×80px、白背景+パディング、`opacity:0.5` で控えめに表示

#### 3.1.2 再生画面（#player）
選択された条文が電光掲示板風にスクロール表示される画面。

##### 横書きモード（デフォルト）
- 画面上部: タイトルバー（現在表示中の条文・項番号を動的表示。例: `004-1`, `004-2`, `前文-1`）
- 画面中央: スクロール表示エリア（電光掲示板風デザイン、右→左スクロール）
- 画面右上: 「一時停止/再生」トグルボタン（⏸ / ▶）＋「設定に戻る」ボタン

##### 縦置きモード（トグルON時）
- ボード: 画面中央に縦長配置（`left:50%; transform:translateX(-50%)`）、上→下方向にスクロール
- タイトル: 縦書き（`writing-mode: vertical-rl`）でボードのすぐ右に配置
- 「設定に戻る」ボタン: 縦書き表示でボードの左側に配置（矢印は「↓」に変更）
- 「⏸」ボタン: 「設定に戻る」の直下に配置
- テキスト: `writing-mode: vertical-rl` で自然な日本語縦書き表示、長体（scaleX）は無効化

### 3.2 スクロール表示仕様

| 仕様 | 内容 |
|------|------|
| スクロール方向 | 横書き: 右から左へ水平スクロール / 縦書き: 上方向へ縦スクロール |
| ループ | 無限ループ（末尾到達で先頭に戻る、シームレス） |
| アニメーション | requestAnimationFrame によるフレーム単位描画 |
| タイトル更新 | 現在表示中の条文・項に応じてタイトルを動的に切り替え（例: `004-2`） |
| 一時停止/再生 | トグルボタンでスクロールの停止・再開を制御 |
| 色サイクル | 複数色選択時、セグメントごとに色を順番に割り当て。条文数×色数のLCM分で1セットとしシームレスループ |
| タイトル色連動 | タイトルバーの色が現在表示中のセグメントの色に連動して変化 |
| 画面回転対応 | 画面回転（縦↔横）時にボード幅・テキスト座標・ボタン位置を自動再計測。スクロール位置も比率補正 |

### 3.3 電光掲示板風デザイン要件

- 背景: 暗色（#080808 / #111）
- 文字: 太字、横書き時は長体90%（`scaleX(0.9)`）、縦書き時は平体90%（`scaleY(0.9)`）、グロー効果（text-shadow による発光表現）
- 走査線: CSS repeating-linear-gradient によるCRTスキャンライン効果
- ボーダー: 上下に電光掲示板のフレームを模した枠線

### 3.4 レスポンシブ対応
- 文字サイズは `vmin` 単位で指定し、画面サイズに自動追従
- 設定画面は `max-width: 480px` でセンタリング
- タッチ操作に対応（ボタン・チェックボックスのタップ領域確保）

## 4. データ構造

### 4.1 条文データ（ARTICLES配列）
JavaScriptの配列としてHTML内にインライン定義。条文は**項単位**に分割する。

```javascript
{
  id: string,            // "preamble-1"〜"preamble-4" | "1-0" | "4-1", "4-2" など
  label: string,         // タイトル表示用（"前文-1" | "001-0" | "004-1", "004-2" など）
  text: string,          // スクロール表示するテキスト本文
  checkbox_label: string // 設定画面のチェックボックス用ラベル（先頭30字、省略表示はCSSに委任）
}
```

#### 4.1.1 ID・ラベル体系

| 種別 | id 例 | label 例 | 説明 |
|------|-------|----------|------|
| 前文 | `preamble-1`〜`preamble-4` | `前文-1`〜`前文-4` | 4段落に分割 |
| 単一項の条文 | `1-0` | `001-0` | 枝番号 `0` で桁数統一 |
| 複数項の条文（第1項） | `4-1` | `004-1` | 第1項は「第○条　〜」をそのまま表示 |
| 複数項の条文（第2項以降） | `4-2` | `004-2` | 「第○条」を割愛し「２　〜」から表示 |

#### 4.1.2 checkbox_label の表示ルール

| 項の種別 | checkbox_label テキスト部分 | 例 |
|----------|---------------------------|-----|
| 全共通 | テキスト本文の先頭30文字（「…」なし）| `001-0:天皇は、日本国の象徴であり日本国民統合の象徴であつて、この地` |
| 第2項以降 | 項番号+スペースを除去した本文の先頭30文字 | `004-2:天皇は、法律の定めるところにより、その国事に関する行為を委任` |

- 画面幅に収まらない場合の末尾省略（`…`）はCSSの `text-overflow: ellipsis` に委任

#### 4.1.3 スクロールテキスト（text）の表示ルール

| 項の種別 | スクロールテキスト | 例 |
|----------|-------------------|-----|
| 第1項 or 単一項 | 「第○条　〜」（条番号付き、従来通り） | `第四条　天皇は、この憲法の定める…` |
| 第2項以降 | 「２　〜」（条番号「第○条」を割愛、項番号から開始） | `２　天皇は、法律の定めるところにより…` |
| 前文 | 段落テキストそのまま | `日本国民は、正当に選挙された…` |

#### 4.1.4 号を持つ条文の扱い
第7条のように「号」（一、二、三…）を持つ条文は、号単位では分割せず1エントリとして扱う。

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 60fps のスムーズなスクロール（requestAnimationFrame使用） |
| 可搬性 | メインHTMLファイル + PWA関連ファイル（manifest.json, sw.js, アイコン）で動作 |
| オフライン | Service Workerによるキャッシュでオフライン利用可能（フォントはブラウザキャッシュに依存） |
| インストール性 | PWA対応によりホーム画面に追加してアプリとして起動可能 |
| アクセシビリティ | タッチ対応、十分なコントラスト比 |

## 6. バージョン管理

- バージョン文字列の形式: `V` + 日付6桁(YYMMDD) + 連番アルファベット（例: `V260327B`）
- 設定画面最下部にバージョン表示（`#222` 色・9px、背景に溶け込む控えめな表示）
- **バージョン更新時は以下の3箇所を必ず同時に更新すること**（片方だけではService Workerのキャッシュが切り替わらず、ユーザーに最新版が配信されない / ビーコンのバージョンが古いままになる）:
  1. `index.html` 内のバージョン表示文字列
  2. `sw.js` の `CACHE_NAME` を同一文字列に更新
  3. `index.html` 内の `app_version` ビーコンの `version` 値を同一文字列に更新

## 7. 制約事項

- サーバーサイド処理なし（完全クライアントサイド）
- 設定値は `localStorage` に永続化（実行ボタン押下時に保存、次回アクセス時に復元）
- 横向き・縦向き両方で使用可能だが、横向き推奨
