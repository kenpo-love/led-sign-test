# 実装仕様書 — 日本国憲法 電光掲示板

## 1. アーキテクチャ概要

単一HTMLファイル（`index.html`）にCSS・JavaScriptをインラインで包含する構成。
外部依存はGoogle Fonts（Noto Sans JP）のみ。PWA対応ファイルを同梱。

```
index.html
├── <head>     … Google Analytics（gtag.js + visibilitychange ビーコン）+ PWAメタタグ（manifest, theme-color, apple-mobile-web-app-*）+ Service Worker登録
├── <style>    … CSS（設定画面 + 再生画面 + 電光掲示板デザイン + インストールバナー）
├── <body>     … HTML（設定画面 #settings + 再生画面 #player + インストールバナー）
└── <script>   … JavaScript（データ定義 + UI生成 + スクロールエンジン + PWAインストールバナー制御）

manifest.json  … PWAマニフェスト
sw.js          … Service Worker（キャッシュファースト戦略）
icon-192.png   … PWAアイコン（192x192）
icon-512.png   … PWAアイコン（512x512）
icon.svg       … アイコン元データ
```

## 2. CSS設計

### 2.1 設定画面
- `.setting-group` : 各設定項目のコンテナ
- `.option-row label` : ラジオボタンをカード風UIに変換（`input[type="radio"]`は`display:none`）
- `:has(input:checked)` : 選択状態のスタイリング（黄色背景）
- `.color-row label` : 色選択は円形ボタン（`border-radius: 50%`）
- `.article-list` : 条文一覧はスクロール可能なリスト（`max-height: 300px; overflow-y: auto`）

### 2.2 再生画面
- `#player` : Flexboxで中央配置（`display: flex; align-items: center; justify-content: center`）
- `.board` : 電光掲示板本体（`overflow: hidden; position: relative`）
  - `::after` 疑似要素でスキャンライン効果を重畳
- `#runner` : テキスト要素を水平に並べる絶対配置コンテナ（`position: absolute; white-space: nowrap`）
- `.seg` : 各条文テキストのスパン要素（`display: inline-block; transform: scaleX(0.9)` で長体90%）

### 2.3 レスポンシブ
- 文字サイズは`vmin`単位 → 画面の短辺に追従
- ボードの高さは `fontSize * 1.3 vmin` で動的設定
- タイトルサイズは `max(5, fontSize * 0.2) vmin`

## 3. JavaScript設計

### 3.1 データ定義

#### ARTICLES 配列
- 日本国憲法の前文（4段落）＋全103条を**項単位**に分割したJSONオブジェクトの配列として定義（164エントリ）
- HTMLファイル内に直接埋め込み（ビルド不要）
- 各エントリは `{ id, label, text, checkbox_label }` の構造

##### ID・ラベル体系
- 前文: `preamble-1`〜`preamble-4` / label: `前文-1`〜`前文-4`
- 単一項の条文: `1-0` / label: `001-0`（枝番号0で桁数統一）
- 複数項の条文: `4-1`, `4-2` / label: `004-1`, `004-2`

##### テキスト生成ルール（text フィールド）
- 第1項 or 単一項: 「第○条　〜」をそのまま格納
- 第2項以降: 「第○条」を割愛し「２　〜」から格納
- 号を持つ条文（第7条など）: 号単位には分割せず1エントリ

##### checkbox_label 生成ルール
- 形式: `{label}:{テキスト先頭30文字}`（末尾の「…」は付与しない）
- 第2項以降: text の先頭の項番号+スペース（例:「２　」）を除去した本文から30文字を切り出す
- 画面幅超過時の省略表示（ellipsis）はCSS側で制御

##### データ生成ツール
- `Dev_研究と改修/260324_条文項分割/generate_articles.py`
- `日本国憲法.txt` を解析し、項単位分割済みのARTICLES配列をJSON出力するPythonスクリプト
- 出力: `Dev_研究と改修/260324_条文項分割/articles.json`

#### COLORS 配列
- 10色を `{ name, value }` で定義
- `name`: 色名（日本語、title属性に使用）
- `value`: CSS色値（スクロールテキスト・ボタン背景色に使用）
- デフォルト選択: 黄（`#ffe600`）

### 3.4 スクロールエンジン（`startScroll` 関数）

#### 引数
```javascript
function startScroll(texts, fontSize, speed, colors, vertical)
// colors: 選択された色の配列（1色以上）
// vertical: 縦書きモードフラグ（boolean）
```

#### 処理フロー
1. **画面切替**: 設定画面を非表示、再生画面を表示（`display: flex`）
2. **サイズ設定**: ボード高さ・タイトルサイズ・パディング（`fontSize * 1.0` = 1文字分）をfontSizeから算出
3. **タイトル初期化**: 最初の条文名をタイトルに設定、最初の色を適用
4. **1セット長の算出**: `lcm(texts.length, colors.length)` で1セットのセグメント数を決定
5. **テキスト・色配列の構築**: `currentTexts[i] = texts[i % texts.length]`, `segColors[i] = colors[i % colors.length]`
6. **テキスト生成**: 1セット分のセグメントを2回分`<span class="seg">`として`#runner`に追加（各セグメントに対応する色を適用）
7. **幅計測**: `document.fonts.ready` 待機後、各セグメントの幅を計測し `segEdges` 配列に累積幅を記録
8. **アニメーション開始**: `requestAnimationFrame(tick)` を呼び出し

#### tick 関数（フレームごとの更新）
```javascript
tickFn = function tick(ts) {
  // 0. paused フラグが true なら即 return（アニメーション停止）
  // 1. デルタタイム算出（上限0.05秒でクランプ）
  // 2. posX -= speed * dt（左方向にスクロール）
  // 3. posX <= -totalW なら posX += totalW（ループ）
  // 4. runner.style.transform = translateX(posX)
  // 5. 現在位置から表示中の条文を特定しタイトル更新
  // 6. requestAnimationFrame(tick) で次フレームを予約
}
```

#### 一時停止/再生の仕組み
- `paused` フラグ（boolean）でスクロール状態を管理
- 一時停止: `tick` 関数先頭で `paused === true` なら `return`（rAFチェーンが自然停止）
- 再生再開: `lastTime = null` でデルタタイムをリセットし `requestAnimationFrame(tickFn)` で再開
- `startScroll()` 呼び出し時に `paused = false` にリセット
- ボタン表示: 再生中 → `⏸`（U+23F8）、停止中 → `▶`（U+25B6）

#### シームレスループの仕組み
- 1セット = `lcm(条文数, 色数)` 個のセグメント（条文と色の全組み合わせが1周する単位）
- 1セット分を2回生成（前半＝本体、後半＝コピー）
- `totalW` は前半（1セット分）の合計幅
- `posX` が `-totalW` を超えたら `+totalW` で巻き戻し
- 色の割り当ても含めて完全にループが一致するため、シームレスな表示を実現

#### タイトル動的更新
- `segEdges[i]` に各エントリの開始位置（累積幅）を記録
- 現在のスクロール位置（`-posX % totalW`）と比較し、該当エントリを特定
- 逆順ループで最後にマッチしたインデックスが現在表示中のエントリ
- タイトルには `currentTexts[i].label` 値を表示
- タイトルの色も `segColors[i]` に連動して毎フレーム更新

### 3.6 縦書きモード

#### 設定UIトグルスイッチ
```html
<button type="button" id="verticalToggle" class="toggle-switch" data-on="false"></button>
```
- CSSのみでスライドアニメーションを実現（`::after` 疑似要素の `translateX` + `transition`）
- `data-on` 属性で状態管理、クリックイベントでトグル
- OFF: グレー背景（`#555`）+ 白丸が左 / ON: 黄色背景（`#ffe600`）+ 白丸が右

#### 縦書き時のCSS切替
`#player.vertical` クラスの付け外しで横書き/縦書きのスタイルを切り替える。

| 要素 | 横書き | 縦書き |
|------|--------|--------|
| `.board` | `width:100%`, 水平帯 | `height:100vh`, 画面中央に縦長配置 |
| `#runner` | `translateX` で左方向スクロール | `translateY` で上方向スクロール |
| `.seg` | `transform: scaleX(0.9)` (長体90%) | `transform: scaleY(0.9)` (平体90%) |
| `.seg` padding | `paddingLeft` | `paddingTop` |
| `.board-title` | 横書き・左上固定 | `writing-mode: vertical-rl`・ボード右側に動的配置 |
| `#btnBack` | 横書き・右上固定 | `writing-mode: vertical-rl`・ボード左側に動的配置 |
| `#btnPause` | 右上固定 | ボード左側、「設定に戻る」直下に動的配置 |

#### startScroll の縦書き対応
```javascript
function startScroll(texts, fontSize, speed, colors, vertical)
```
- 第5引数 `vertical` (boolean) を追加
- `isVertical` グローバル変数で状態保持
- 縦書き時の分岐:
  - ボードサイズ: `height: 100vh`, `width: fontSize * 1.5 vmin`
  - セグメント padding: `paddingTop` を使用
  - tick関数: `translateY(posX)` でスクロール
  - 幅計測: `offsetHeight` を使用

#### ボタン位置の動的計算
`document.fonts.ready` 後にボードの `getBoundingClientRect()` で実際の位置を取得し、条文タイトル・ボタンの位置を動的に設定。横書きに戻る際は全てデフォルト位置にリセット。

### 3.7 画面回転対応（recalcLayout）

#### recalcLayout() 関数
`document.fonts.ready` 内の計測・配置ロジックを関数として抽出。初回計測時および画面回転時に呼ばれる。

```javascript
function recalcLayout() {
  // 0. scaleX/Y(0.9) 分のネガティブマージンを各セグメントに適用
  //    → transform は視覚のみ縮小しレイアウト幅は不変のため、
  //      margin-right（縦書き時は margin-bottom）= -(offsetWidth * 0.1) で補正
  // 1. 全セグメントの幅/高さを再計測し totalW, segEdges を再構築
  //    → totalW += offsetWidth + margin（マージン込みの実効幅）
  // 2. posX を新旧 totalW の比率で補正（表示位置のジャンプ防止）
  // 3. 縦書き時のボタン・タイトル位置を再計算
}
```

#### イベントリスナー
```javascript
// Chrome/Edge 等: resize イベント（150ms デバウンス）
window.addEventListener('resize', onResize);

// Safari 対応: orientationchange イベント（3段階遅延再計測）
window.addEventListener('orientationchange', onOrientationChange);
// → 100ms / 300ms / 600ms で recalcLayout() を呼び出し
// → Safari は回転アニメーション中に不正確なサイズを返すため段階的に再計測
```

#### スクロール位置の補正
```javascript
// 回転前後で totalW が変わるため、posX を比率補正
if (oldTotalW > 0 && totalW > 0) {
  posX = posX * (totalW / oldTotalW);
}
```

## 4. パフォーマンス考慮

| 手法 | 目的 |
|------|------|
| `will-change: transform` | GPU レイヤー昇格によるスムーズなスクロール |
| `translateX()` / `translateY()` のみ使用 | レイアウト再計算を回避（Compositor-only animation） |
| `.seg` に `scaleX(0.9)` / `scaleY(0.9)` | 横書き長体90% / 縦書き平体90%を各セグメント個別に適用。`offsetWidth`/`offsetHeight`（レイアウト幅）で計測し、ネガティブマージン（`-10%`）でレイアウト幅と視覚幅の乖離を補正 |
| `recalcLayout()` の軽量性 | DOM読み取り+数値計算のみ（1ms未満）。resize/orientationchange で複数回呼んでもフレーム落ちしない |
| `dt` のクランプ（上限0.05秒） | タブ非アクティブ復帰時のジャンプ防止 |
| `document.fonts.ready` 待機 | フォントロード完了後に正確な幅計測を実行 |
| タイトル更新を `currentSegIdx` で制御 | セグメント変化時のみDOM更新し、毎フレームのDOM操作を回避 |
| `text-shadow` 2段（`glow()` 関数） | 3段→2段に削減し描画負荷を軽減 |

### 4.1 設定の永続化（localStorage）

#### saveSettings()
実行ボタン押下時に全設定値をJSON化して保存。
```javascript
function saveSettings() {
  const settings = {
    size: ...,      // ラジオボタンvalue
    speed: ...,     // ラジオボタンvalue
    colors: [...],  // チェック済み色値の配列
    articles: [...],// チェック済み条文IDの配列
    freeMsg1: ...,  // テキスト
    freeMsg2: ...,  // テキスト
    vertical: ...   // "true" / "false"
  };
  localStorage.setItem('kenpou-settings', JSON.stringify(settings));
}
```

#### restoreSettings()
ページ読み込み時に保存済み設定を復元。`try/catch` で囲み、保存データが破損していても安全にフォールバック。
- ラジオボタン: `.checked = true` で選択状態を設定
- チェックボックス: `.checked = includes(value)` で一括復元
- テキスト入力: `.value` に直接代入
- トグルスイッチ: `.dataset.on` を設定

### 7.3 操作イベントビーコン（start_scroll / free_message）

実行ボタン押下時に、選択された条文とフリーメッセージの内容をカスタムイベントとして送信する。

#### start_scroll イベント
```javascript
gtag('event', 'start_scroll', {
  articles: articleLabels.slice(0, 100),  // 選択条文のラベルをカンマ区切り（上限100文字）
  article_count: checked.length,          // 選択された条文数
  font_size: fontSize,                    // 数値: 25 / 40 / 55
  font_size_label: ({25:'小',40:'中',55:'大'})[fontSize],
  speed: speed,                           // 数値: 160 / 300 / 500
  speed_label: ({160:'緩',300:'中',500:'速'})[speed]
});
```
- `articles`: 選択された条文の `label` をカンマ区切りで連結（例: `"前文-1,第9条,第25条"`）。GA4パラメータ上限の100文字で切り捨て
- `article_count`: 選択された条文の数（数値）
- `font_size` / `speed`: ラジオボタンの `value`（ピクセル数・px/秒）を数値のままイベント指標として送信。GA4で平均値・分布を集計するために使用
- `font_size_label` / `speed_label`: 小/中/大・緩/中/速 の日本語ラベル。GA4レポートで人間が読める単位で分類表示するために使用

#### free_message イベント
```javascript
if (freeText1 !== '' && freeText1 !== lastSentMsg1) {
  gtag('event', 'free_message', { message_text: freeText1.slice(0, 100) });
  lastSentMsg1 = freeText1;
}
```
- メッセージ欄①②それぞれについて、入力がある場合のみ送信（空欄はスキップ）
- `message_text`: 入力テキストを100文字以内で送信
- **重複送信防止**: `lastSentMsg1` / `lastSentMsg2` に前回送信したメッセージを記憶し、同一セッション内で同じ内容の場合は送信しない。ページリロード時はリセットされる

#### GA4管理画面側の設定
以下を「管理」→「カスタム定義」で登録する必要がある。

| 表示名 | イベントパラメータ | 種別 | 範囲 |
|---|---|---|---|
| 選択条文 | articles | ディメンション | イベント |
| 条文数 | article_count | 指標（整数） | イベント |
| フリーメッセージ | message_text | ディメンション | イベント |
| 文字サイズ | font_size | 指標（整数） | イベント |
| 文字サイズラベル | font_size_label | ディメンション | イベント |
| スクロール速度 | speed | 指標（整数） | イベント |
| 速度ラベル | speed_label | ディメンション | イベント |

### 7.4 PWAバックグラウンド復帰時のビーコン
```javascript
document.addEventListener('visibilitychange', function() {
  if (document.visibilityState === 'visible') {
    gtag('event', 'page_view', { page_title: document.title });
  }
});
```
- PWAがバックグラウンドから復帰した際（ホームに戻って再度開く、画面ロック解除など）にビーコンを送信
- ページのリロードは発生しないため、localStorage の設定値は保持される
- gtagスクリプト自体はService Workerのキャッシュ対象外のため、WiFi接続時のみ送信される（オフライン時は静かに失敗）
